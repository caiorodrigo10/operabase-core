# Create PRD

> Task for @pm (Morgan) | Product

## Purpose

Create a Product Requirements Document for a new feature or product.

## Prerequisites

- Problem statement or feature request
- Stakeholder input
- Market context

## Workflow

### Phase 1: Discovery

1. Understand the problem being solved
2. Identify target users
3. Research existing solutions
4. Define success metrics

### Phase 2: Requirements

1. Define functional requirements
2. Define non-functional requirements
3. Create user stories overview
4. Define scope (in/out)
5. Identify risks and assumptions

### Phase 3: Documentation

1. Write PRD using project template
2. Include wireframes or mockups if available
3. Define acceptance criteria
4. Set priorities (MoSCoW)

### Phase 4: Review

1. Review with stakeholders
2. Incorporate feedback
3. Get sign-off

## Outputs

- PRD document in docs/
- User stories outline
- Success metrics defined
