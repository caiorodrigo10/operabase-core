# Performance Review

> Task for @qa (Quinn) | Performance

## Purpose

Review code for performance issues and optimization opportunities.

## Prerequisites

- Code to review
- Performance criteria or benchmarks

## Workflow

### Phase 1: Analysis

1. Identify hot paths and critical operations
2. Check for N+1 queries
3. Review memory allocation patterns
4. Check for unnecessary re-renders (frontend)
5. Review async/await patterns

### Phase 2: Measurement

1. Profile critical paths
2. Measure response times
3. Check bundle size impact
4. Review database query plans

### Phase 3: Recommendations

1. Document performance issues found
2. Suggest optimizations with expected impact
3. Prioritize by effort vs impact

## Outputs

- Performance review report
- Optimization recommendations
- Before/after metrics (if changes applied)
