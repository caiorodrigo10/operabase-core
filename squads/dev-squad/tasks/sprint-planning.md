# Sprint Planning

> Task for @sm (River) | Process

## Purpose

Facilitate sprint planning session and organize work for the next sprint.

## Prerequisites

- Backlog is groomed and prioritized
- Team capacity is known
- Previous sprint retrospective completed

## Workflow

### Phase 1: Preparation

1. Review backlog priorities with @po
2. Check velocity from previous sprints
3. Identify dependencies between stories
4. Note any blocked items

### Phase 2: Planning

1. Select stories for the sprint based on priority and capacity
2. Break stories into tasks if needed
3. Identify risks and dependencies
4. Set sprint goal

### Phase 3: Commitment

1. Confirm team capacity
2. Present sprint plan for approval
3. Document sprint backlog
4. Create sprint board

## Outputs

- Sprint backlog (selected stories)
- Sprint goal
- Identified risks/dependencies
