# Create Component

> Task for @dev (Dex) | Frontend

## Purpose

Create a new UI component following project design system and conventions.

## Prerequisites

- Component specification (props, behavior, visual design)
- Design system tokens available
- Framework setup (React, Vue, Svelte, etc.)

## Workflow

### Phase 1: Design

1. Review existing component patterns
2. Define component interface (props/API)
3. Identify state management needs
4. Plan accessibility requirements

### Phase 2: Implementation

1. Create component file following naming conventions
2. Implement component structure
3. Add styling (CSS modules, Tailwind, styled-components)
4. Add event handlers and state logic
5. Ensure responsive behavior

### Phase 3: Testing

1. Write unit tests
2. Test accessibility (keyboard navigation, screen reader)
3. Test responsive behavior
4. Visual review

## Outputs

- Component file
- Style file (if separate)
- Test file
- Story file (if Storybook used)

## Validation

- [ ] Component renders correctly
- [ ] Props API is clean and documented
- [ ] Accessible (ARIA, keyboard nav)
- [ ] Responsive
- [ ] Tests pass
